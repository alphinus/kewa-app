# Requirements: KEWA Renovation Operations System

**Defined:** 2026-01-22
**Milestone:** v2.1 Master Data Management
**Core Value:** KEWA AG hat volle Transparenz und Kontrolle uber alle Renovationen

## v2.1 Requirements

Requirements for v2.1 Master Data Management. Each maps to roadmap phases.

### Partner/Handwerker (PART)

- [x] **PART-01**: Admin kann Partner-Liste mit allen Handwerkern anzeigen
- [x] **PART-02**: Admin kann neuen Partner erstellen (Firma, Kontakt, Email, Gewerke)
- [x] **PART-03**: Admin kann Partner-Daten bearbeiten
- [x] **PART-04**: Admin kann Partner deaktivieren/aktivieren
- [x] **PART-05**: Partner-Dropdown in WorkOrderForm zeigt verfuegbare Partner

### Liegenschaften (PROP)

- [x] **PROP-01**: Admin kann Liegenschaft-Liste mit allen Properties anzeigen
- [x] **PROP-02**: Admin kann neue Liegenschaft erstellen
- [x] **PROP-03**: Admin kann Gebaeude zu Liegenschaft hinzufuegen
- [x] **PROP-04**: Admin kann aktive Liegenschaft im Header wechseln
- [x] **PROP-05**: Dashboard/Heatmap zeigt gewaehlte Liegenschaft

### Einheiten (UNIT)

- [x] **UNIT-01**: Admin kann Wohnungen/Einheiten zu Gebaeude hinzufuegen
- [x] **UNIT-02**: Admin kann Einheit-Details bearbeiten (Name, Etage, Mieter)
- [x] **UNIT-03**: Admin kann Raeume zu Einheit hinzufuegen
- [x] **UNIT-04**: Raeume werden fuer Condition-Tracking verwendet

### Templates (TMPL)

- [x] **TMPL-01**: Admin kann Template-Liste mit allen Vorlagen anzeigen
- [x] **TMPL-02**: Admin kann Template-Details ansehen (Phasen, Pakete, Tasks)
- [x] **TMPL-03**: Admin kann neues Template erstellen
- [x] **TMPL-04**: Admin kann Template bearbeiten (Phasen/Pakete/Tasks hinzufuegen)
- [x] **TMPL-05**: Admin kann Template beim Projekt-Erstellen auswaehlen

### Admin-Dashboard (ADMN)

- [x] **ADMN-01**: Uebersichts-Dashboard zeigt Zaehler (Liegenschaften, Partner, Projekte)
- [x] **ADMN-02**: Quick-Actions fuer haeufige Admin-Tasks
- [x] **ADMN-03**: Such-/Filterfunktion auf Partner- und Template-Listen

### Onboarding (SEED)

- [x] **SEED-01**: Migrations 045, 046 sind committed und dokumentiert
- [x] **SEED-02**: Demo-Daten-Script fuer Testzwecke
- [x] **SEED-03**: Setup-Wizard beim ersten Login (Liegenschaft, Gebaeude, Partner)
- [x] **SEED-04**: README mit Deployment-Anleitung

## Out of Scope

Explicitly excluded for v2.1.

| Feature | Reason |
|---------|--------|
| Tenant Portal | v3.0 scope — MVP focuses on admin/internal users |
| Offline Support | Complex PWA implementation deferred |
| External Integrations | Calendar/accounting integrations deferred |
| Template Import/Export | Nice-to-have, not critical for v2.1 |
| Bulk Partner Import | Single-entry sufficient for initial rollout |
